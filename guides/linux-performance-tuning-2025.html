Linux Performance Tuning Guide (2025)

Use these tweaks after you install a performance-oriented distro such as Clear Linux, Arch, or EndeavourOS. Snapshot first and change one variable at a time.

1. Sysctl profile

Create /etc/sysctl.d/95-performance.conf with workstation-friendly defaults:

vm.swappiness=20
vm.dirty_ratio=10
vm.dirty_background_ratio=5
kernel.sched_min_granularity_ns=10000000
kernel.sched_wakeup_granularity_ns=15000000
net.core.default_qdisc=fq
net.ipv4.tcp_congestion_control=bbr

BBR plus the fq qdisc dramatically improve latency on fiber and Wi-Fi 6 networks. Apply with sudo sysctl --system and document the change inside your homelab notes.

2. Swappiness & zram

Keep real RAM hot by lowering swappiness and enabling compressed swap:

sudo systemctl enable --now systemd-zram-setup@zram0.service

Rolling distros such as Arch and EndeavourOS expose zram-generator. Configure zram-size = ram / 2 on laptops and set vm.swappiness=15 for workstation desktops.

3. Journald rate limiting

Chattier services can stall writes. Tune /etc/systemd/journald.conf:

[Journal]
SystemMaxUse=750M
RateLimitIntervalSec=20s
RateLimitBurst=400

Restart journald, then forward critical logs with systemd-journal-upload or ship to Loki/Elastic. This mirrors the logging setup we recommend on the Kernel Update Guide.

4. Filesystem-specific wins

Filesystem Tweak Recommended For
EXT4 sudo tune2fs -o journal_data_writeback /dev/<root> Desktop installs that still want classic journaling. Pair with weekly fstrim.
Btrfs Mount with compress=zstd,ssd,space_cache=v2 Snapshot-heavy distros like Garuda or EndeavourOS.
XFS Increase log buffer via logbsize=256k Servers running RHEL or AlmaLinux handling big sequential writes.
ZFS Set recordsize=1M for media workloads Workstations experimenting with Pop!_OS ZFS or Clear Linux storage pools.

5. Trim background services

Run systemctl --type=service --state=running to find daemons you can disable. Examples: printing on travel laptops, media indexers on render nodes, or tracker miners if you already index with Syncthing. Remove unused stacks cleanly with help from the Package Manager Cheatsheet.

6. Benchmark and iterate

  • systemd-analyze blame highlights services slowing boot.
  • fio --name=randread --rw=randread captures storage deltas.
  • perf stat make -j$(nproc) surfaces CPU scheduling impact.

Record the kernel build and distro version (e.g., Clear Linux build number) with every change. Timeshift or Btrfs snapshots let you roll back when a tune-up backfires.
