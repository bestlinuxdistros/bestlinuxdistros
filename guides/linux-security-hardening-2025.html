Security & Compliance

Linux Security Hardening Guide (2025)

This playbook distills how we lock down lab machines that run Kali, Parrot, and Qubes OS. Adapt it to any distro in the security category of our catalog.

1. Start with a sane firewall baseline

Most distros ship either ufw or firewalld. Pick one and enforce default deny:

sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow OpenSSH
sudo ufw enable

On Fedora-based systems (including Qubes templates) use firewall-cmd services and zones. For penetration-testing rigs like Kali Linux, create profiles so scanners run inside isolated namespaces rather than on the workstation host network.

2. Harden the kernel

Apply sysctl toggles that reduce exploit surface:

kernel.kptr_restrict=2
kernel.dmesg_restrict=1
kernel.yama.ptrace_scope=2
net.ipv4.conf.all.accept_redirects=0
net.ipv4.conf.all.accept_source_route=0
fs.protected_symlinks=1

Drop them inside /etc/sysctl.d/99-hardening.conf and run sudo sysctl --system. Distros such as Parrot Security already set many of these values, but auditing ensures upgrades don’t revert the policy.

3. Tighten permissions

  • Ensure only root writes to /boot.
  • Set the sticky bit on shared directories: sudo chmod 1777 /tmp /var/tmp.
  • Use find / -perm -4000 -type f to inventory SUID binaries and document why they exist.
  • Adopt role-based sudoers files so pentest tooling is separated from daily-driver accounts.

We mirror this workflow inside our catalog when evaluating whether a distro belongs in the Security tag set.

4. Embrace SELinux and AppArmor

Qubes OS uses Xen isolation, but even mainstream distros gain guardrails with MAC frameworks:

  • SELinux (Fedora, RHEL derivatives): Set to enforcing, install setroubleshoot, and tailor boolean toggles instead of disabling the policy.
  • AppArmor (Ubuntu, Debian-based): Enable profiles for browsers and containers, then review with aa-status.

If you build golden images, store custom profiles in Git and reapply them with Ansible so laptops stay in sync.

5. Monitor and log intelligently

Centralized logging matters even for single engineers. Forward journald to text files with rate limiting:

[Journal]
RateLimitIntervalSec=30s
RateLimitBurst=500

Ship the logs with systemd-journal-upload or vector. Pair that with file integrity monitoring via auditd or osquery.

6. Pick the right distro for the job

Match your workflow to the distro profile:

  • Kali Linux: Offensive tooling with rolling releases. Use snapshots or kernel update guardrails before major upgrades.
  • Parrot Security OS: Includes sandboxed applications and a lighter footprint for laptops.
  • Qubes OS: Xen-based compartmentalization when you need to isolate identities or customers.

Cross-reference their hardware requirements on each detail page’s Hardware section or inside the compatibility tool.
