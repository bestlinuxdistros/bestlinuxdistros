Linux Kernel Update Guide

Kernel updates keep hardware secure and performant, but every distro handles them differently. Use this reference before you upgrade Arch, Manjaro, Ubuntu, or Fedora.

1. Know your cadence

  • LTS distros (Ubuntu LTS, Debian): kernels land via point releases or Hardware Enablement stacks. They’re conservative but require reboots on your schedule.
  • Rolling distros (Arch, Clear Linux): kernels arrive weekly. Expect occasional driver regressions and keep fallback entries in your bootloader.
  • Managed rolling (Manjaro, Pop!_OS): updates are delayed slightly for extra testing.

2. Snapshot before touching kernels

Snapshot Tool Recommended Distros Command
Timeshift (rsync/Btrfs) Ubuntu, Mint, Pop!_OS sudo timeshift --create --comments "pre-kernel"
Snapper openSUSE, Btrfs-based Arch spins sudo snapper create -d "before kernel"
Refind/btrbk snapshots Arch, Garuda, EndeavourOS sudo btrbk run

3. Execute the upgrade

  • Arch/EndeavourOS: sudo pacman -Syu, watch for hooks, and reboot immediately. Keep linux-lts installed as backup.
  • Manjaro: Use sudo pacman -Syu or the GUI Kernel manager. Pause if the forum sticky flags issues.
  • Fedora: sudo dnf upgrade --refresh or enable akmods if you need Nvidia modules rebuilt.
  • Ubuntu/Mint: Install HWE kernels via sudo apt install --install-recommends linux-generic-hwe-24.04.

4. Rollback tactics

  1. Select the previous kernel from GRUB (Shift key at boot) or from systemd-boot’s menu.
  2. If the distro still boots, remove the bad kernel: sudo pacman -R linux (keeping linux-lts) or sudo apt remove linux-image-6.8.0-xx.
  3. Restore from snapshots via Timeshift/Snapper if the bootloader breaks.

5. Document everything

Track kernel versions, module rebuild commands, and driver tweaks in a simple markdown file. Reference it when tuning with the Performance guide or when you revisit the compare matrix.
